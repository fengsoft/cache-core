CacheCore docs are organized by purpose:

- `concepts/`: keys, policies, tags, invalidation
- `architecture/`: library, adapter, admin API boundaries
- `reference/`: OpenAPI and package reference
- `guides/`: adopting CacheCore in a service
- `operations/`: warming, invalidation, incident response
- `adr/`: architecture decisions
